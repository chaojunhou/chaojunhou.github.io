这篇文章主要关注CPP下实用的编程技术，包括异常处理，防御性编程，模版编程，并发编程，以及设计CPP的设计模式等。

异常处理

C语言处理错误方法通常有三种：

  • 在函数中返回错误信息，如果无法返回，则设置一个全局的错误变量标志（errno，perror（））
  • C语言中的信号函数（signal（））和处理函数（raise())
  • 使用标准C库中的非局部跳转函数，setjump和longjump，setjump可以保存在程序中保存一个已知的无错误的状态，一旦发生错误就可以调用longjump返回到原来的状态。

捕获所有异常

可以使用省略号代替异常处理器的参数来捕获所有类型的异常。

catch(...){
    doSomething();
}

清理

C++的异常处理必须确保当程序的执行程序离开一个作用域的时候，对于属于这个作用域的所有由构造函数建立起来的对象，它们的析构函数一定会被调用。但是如果一个对象的构造函数在执行过程中抛出异常，那么这个对象的析构函数就不会被调用，为了防止在构造函数中造成的资源泄露问题，可以使用下面两种方式之一来解决这个问题： 1. 在构造函数中捕获异常，用于释放资源 2. 在对象的构造函数中分配资源，并在在对象的析构函数中释放资源。

异常规格说明

void f() throw(toobig, toosamll) // 抛出指定异常
void f() throw()//throw中没有参数，则表明不抛出任何异常

模版编程

A template implies an interface take any type

typename关键字

当模版中出现一个标识符时，编译器如果可以在把这个标识符当作一个类型，或者把它当作一个除类型之外的其他元素之间进行选择， 那么编译一般不会认为这个标识符是一个类型，一般情况，它会把标识符当作一个对象，或者是枚举。除非是有typenmae关键字出现，才会把它当作一个嵌套类型。

模版特化

为了解决函数模版重载的问题,系统为这些模版定义了次序,在生成模版函数的时候,编译器将从这些函数中选择特化程度最高的模版。但是函数模版并不支持特化。模版的特化一般是对类模版而言的。

显式特化

前缀"template <>"告诉编译器接下来是一个模版特化,模版特化的类型必须出现在函数名货紧跟的尖括号中,就像通常一个明确指定参数类型的函数调用中一样。

// vector<bool>基本模版的声明
template<class T, class Allocator = allocator<T> >
class vector {...};
// bool类型的特化
template<> class vector<bool, allocatro<bool> > {};

半特化

//vector<bool>声明
template<class Allocator> class vector<bool, Allocator>;

template后面和class关键字后面的尖括号里都是非空的参数列表。这样用户就可以自定义allocator的类型，构成了一种重载的类模版。

模版名字查找问题

当编译器看到一个标识符，它必须能够确定这个标识符所代表的type和作用域（如果是变量就是lifetime）。然而当编译器看到模版的时候，在模版没有实例化之前，是无法知道更多的信息的。这种情况导致了模版编译需要分两个阶段进行。

模版中的name

在第一阶段，编译器解析模版定义，寻找明显的语法错误，resolve所有的name。对于不依赖与模版参数的名称， 编译器使用普通name的查找方法解析它们，如果有必要，编译器也会依赖模版参数进行查找。因此模版编译的第二个阶段就是模版实例化，在这里由编译器决定是否使用模版的一个显式特化来取代基本的模版。

The Curiously recurring template pattern

# include <iostream>
using namespace std;

template<class T>  class Counted{
    static int count;
public:
    Counted(){++count;}
    Counted(const Counted<T>&){++count;}
    ~Counted(){--count;}
    static int getCount() {return count;}   
};

template <class T> int Counted<T>::count = 0;
class CountedClass : public Counted<CountedClass>{};
class CountedClass2 : public Counted<CountedClass2>{};



int main() {
    CountedClass a;
    cout << CountedClass::getCount()<<endl; // 1
    CountedClass b;
    cout<< CountedClass::getCount()<<endl; // 2
    CountedClass2 c;
    cout << CountedClass2::getCount()<<endl; // 1
}

每个派生类都派生于一个唯一的基类，这个基类被用于作为一个模版的参数。由于Counted的数据成员不依赖于T，当模版被解析的时候，Counted的大小（为零）就可以知道，因此使用使用什么参数来实例化Counted无关紧要。当它被解析的时候，用任意一个Counted实例的派生类当然也可以完成，而且不会产生递归。由于每个基类都是唯一的，都有属于自己的静态数据，因此这样就可以实现向任何类中增加计数的便捷方法。

模版元编程

模版元编程就是一种支持编译时编程的机制，它可以很方便的在编译时期实现类型参数的substitiution工作。

enum关键字
  • 用enum关键字说明的常量由编译程序自动生成，程序要不需要用手工对常量一一赋值。
  • 用enum关键字说明常量使程序更清晰易读，因为在定义enum常量的同时也定义了一个枚举类型标识符。
  • 在调试程序时通常可以检查枚举常量

模版编译类型

为了减少包含类型所需要的大的头文件，C++提供了两中可供选择的代码组织机制，即explict instantiation和exported templates（支持最大限度的独立的编译）。

explict instantiation

当显式的实例化一个类时，除了以前已经实例化的成员外，这个特化类的所有成员函数都被实例化。而隐式实例化只有被调用的成员函数才进行实例化。

显式实例化常用于大型软件工程项目中，因为这样可以避免大量的编程时间。

export

在需要在声明中加入export关键字即可，但是目前支持这样实现的编译器不多。

设计模式

设计模式的目的就是为了封装用户的change，最大限度的减少可能用户代码改变带来的副作用。

设计模式可以分为三类，Creational，Structural，Behavioral。

Cretional

一个对象是如何被创建的，这设计到将对象创建的细节和隔离出来，这样你的代码就不依赖于具体对象的type，而且增加一个新的类型的对象的时候也不用去改变原有的代码。比较常用的是单例模式，工厂模式和Builder模式。

Structural

这影响着对象之间连接的方式，使得系统的改变不会影响对象之间的connections。比较常用的是代理模式和适配器模式。

Behavioral

行为模式就是对象如何handle程序中特定type的方式，主要是你想要实现的一些封装的处理， 比较常用的是观察者模式，Template 方法， 命令模式，依赖链模式，Vistor模式。

Singleton

使用奇异递归模版实现的单例模式，MyClass在初始化的时候会先初始化Singleton，然后创建一个只属于自己的类，并进行初始化，由于Singleton模版中的拷贝和赋值构造函数都是私有的， 因此MyClass从被实例化时就有且只有一个实例。

# include<iostream>
using namespace std;

template <class T>  class Singleton{
    Singleton(const Singleton&);
    Singleton& operator = (const Singleton&);
protected:
    Singleton() {
    }
    virtual ~Singleton(){
    }
public:
    static T& instance() {
        static T s;
        return s;
    }

};


class MyClass : public Singleton <MyClass>{
    int x;
protected:
    friend class Singleton<MyClass>;
    MyClass () {x=47;
    }
public:
    void setValue(int i) { x = i;
    }
    const int getValue() {
        return  x;
    }
};




int main() {
    MyClass& m = MyClass::instance();
    cout << m.getValue()<<endl;
    m.setValue(54);
    cout << m.getValue()<<endl;
    return 1;
}

Command

Command就是把一个函数封装为一个object，然后这个object作为一个参数，传递给另一个函数和object。Command is a Messenger that carries behavioral。

# include<iostream>
# include<vector>

using namespace std;

class Command{
    public:
        virtual void execute () = 0; // abstract base class
};

class Hello:public Command{
    public:
        void execute() {
            cout<<" Hello ";
        }
};

class World:public Command{
    public:
        void execute() {
            cout<< " Word ";
        }
};

class IAm:public Command{
    public:
        void execute() {
            cout << " I Am the command pattern!"<<endl;
        }
};


class Macro{
    vector<Command*> commands;

public:
    void add(Command* c) {
    commands.push_back(c); 
}
    void run() {
    for( auto it = commands.begin() ; it!=commands.end(); ++it){
        (*it)->execute();   
    }   
    }
};




int main() {
    Macro macro;
    macro.add(new Hello);
    macro.add(new World);
    macro.add(new IAm);
    macro.run();
    return 0;
}

Command模式通常被使用于Undo中，它可以把undo的操作放到一个队列中，然后逐步的去完成这些操作。
